Pluggable provider seam for the browser card/GPIO layer

Refactors the browser VM (vm/picovm.js) so the card store and GPIO behind
Storage.*/Gpio.* are injectable, per the device-bindings design: the editor (and
PIOS adapters like PiosCapsuleStore) can supply their own backend without forking
the VM.
- new PicoVM({ cardStore, gpioProvider }): cardStore is any PicoStore-compatible
  CRUD+query object; gpioProvider is { pins: {}, count: N } the program drives and
  the editor renders (so the GPIO visual observes pin state live).
- Defaults are the built-in reference store + emulator, so VM parity is unchanged
  (full suite green, incl. test_devices Python==JS, test_store, test_runtime_storage).
The Python VM stays the reference oracle (unchanged). Regenerated playground.html.
